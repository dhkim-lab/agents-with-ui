## 1. 프로젝트 개요

### 1.1 한 줄 정의

사용자가 마케팅 목표를 입력하면, **AI 에이전트 5명이 실제 팀처럼 루틴 업무를 중단하고 프로젝트에 투입되어 회의·작업·판단·납품하는 전 과정**을 2D 픽셀아트 오피스 UI로 실시간 관람하는 교육용 인터랙티브 데모.

### 1.2 사용 맥락

- **대상:** LG 인화원 팀장 승진 예비 핵심인재 13명 (구매사업가 AX 워크샵)
- **시연 시점:** 2026.04.08~09 (1박2일) 세션 4 라이브 데모
- **목적:** "AI Agent가 팀 하나를 꾸려서 실제로 일할 수 있다"는 것을 시각적으로 증명
- **시연 환경:** 프로젝터 + 16:9 모니터 풀스크린

### 1.3 핵심 가치

| 가치 | 설명 |
|------|------|
| 🏢 상시 팀원 감각 | 에이전트가 루틴 업무를 하다가 프로젝트에 투입 → "일회용 도구"가 아닌 "팀원" |
| 🤝 Human-in-the-loop | 중간에 사람이 방향을 정하면 결과가 달라지는 것을 체감 |
| 🧠 자체 판단 디테일 | "이런 것까지 고민해?" — 데이터 신뢰성, 로컬 시장 맥락, 예산 효율 자체 판단 |
| 📦 실체 있는 산출물 | 채팅 로그만이 아닌, 실제 표·차트·카피가 화면에 렌더링 |
| ⚡ 속도 & 비용 시각화 | 사람 팀 4일 vs AI 팀 ~3분, 비용 $0.02~0.03 |

### 1.4 언어 규칙

- **에이전트 이름:** 영어 유지 (Alex, Kai, Mia, Nova, Rex)
- **대화·산출물:** 전부 한국어
- **UI 라벨:** 영어 (Phase, Status, Dashboard 등)

## 3. 16:9 풀스크린 레이아웃

### 3.1 레이아웃 구조 (1920×1080 기준)

```
┌─────────────────────────────────────────────────────────────────────┐
│ TopBar (40px)                                                       │
│ Phase명 │ 경과시간 │ 토큰비용 │ 사람팀 4일 vs AI팀 XX초 │ 🔔 알림  │
├─────────────────────────────────┬───────────────────────────────────┤
│                                 │                                   │
│  OfficeScene (픽셀아트 오피스)    │  ChatLog (세로 풀)                 │
│  에이전트 아바타 + 걷기 동선       │  대화 + 자체판단 + 루틴 알림         │
│  말풍선(statusText)             │  키워드 컬러링 (에러/성공/판단)       │
│  ~58% 너비, ~55% 높이           │  ~27% 너비, ~100% 높이             │
│                                 │  (세로 전체 사용 — 스크롤 맥락 유지)  │
├─────────────────────────────────┤                                   │
│                                 │                                   │
│  ArtifactGallery (산출물 갤러리)  ├───────────────────────────────────┤
│  에이전트별 탭 + 실시간 렌더링     │  StatusPanel (컴팩트)              │
│  표·차트·카피 카드              │  5명 상태 (루틴/프로젝트 라벨)       │
│  ~58% 너비, ~45% 높이           │  ~27% 너비                        │
│                                 │  ~35% 높이                        │
├─────────────────────────────────┴───────────────────────────────────┤
│ BottomBar (48px) — 에이전트 미니 프로필 5명 (아바타+이름+현재작업)      │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 비율 가이드 (1920×1080 → 콘텐츠 영역 1920×992px)

| 영역 | 너비 | 높이 | 비고 |
|------|------|------|------|
| TopBar | 100% | 40px | 고정 |
| OfficeScene | 58% (1114px) | 55% (~546px) | 좌상단 |
| ArtifactGallery | 58% (1114px) | 45% (~446px) | 좌하단 |
| ChatLog | 27% (518px) | 65% (~645px) | 우상단, 세로 풀 |
| StatusPanel | 27% (518px) | 35% (~347px) | 우하단 |
| BottomBar | 100% | 48px | 고정 |
| (나머지 15%) | 갭/패딩 | — | 좌우 gap 16px × 2 + 중간 gap 16px |

### 3.3 프로젝터 시연 시 주의사항

- ChatLog 폰트 최소 14px (뒤에서도 읽을 수 있도록)
- OfficeScene 에이전트 크기 최소 48×48px
- Decision Popup은 화면 중앙 오버레이, 최소 너비 480px

| 컴포넌트 | 변경사항 |
|----------|----------|

| **StatusPanel** | 우하단 컴팩트. 루틴/프로젝트 라벨 추가. 5명 동시 표시 (폰트/패딩 다이어트) |

### 4.2 신규 컴포넌트

| 컴포넌트 | 설명 |
|----------|------|
| **ArtifactGallery** | 좌하단. 에이전트별 탭 (Kai/Nova/Mia/Rex/통합). 작업 완료 시 산출물이 탭에 실시간 렌더링. Kai→경쟁사 분석표, Nova→퍼널 다이어그램, Mia→카피 3종 카드, Rex→예산 배분 차트+ROI 테이블 |
| **DecisionPopup** | 화면 중앙 오버레이. Alex의 질문 + 선택지 2~3개 + 30초 카운트다운(원형 프로그레스). 알림음 재생. 타임아웃 시 자동 선택 + 근거 설명 |
| **BottomBar** | 하단 고정. 에이전트 5명 미니 프로필 (픽셀 아바타 축소 + 이름 + 현재 작업 한 줄). 루틴/프로젝트 상태 표시 |
| **PixelAvatar** | 2D 픽셀아트 캐릭터. 스프라이트 시트 기반 애니메이션 (걷기 4방향 + 앉기 + 타이핑 + idle). OfficeScene 내부에서 사용 |
| **OfficeMap** | 아이소메트릭 또는 탑다운 타일맵. 책상 5개 + 회의실 + 복도. 경로 기반 에이전트 이동 |

### 4.3 이벤트 타입 레퍼런스

| 타입 | 설명 | 주요 파라미터 |
|------|------|---------------|
| `chat` | 채팅 메시지 | `agent`, `text`, `tag?` (normal/error/success/insight) |
| `move` | 위치 이동 | `agent` (개별 or `'all'`), `to` (desk/meeting_room/Alex_desk) |
| `status` | 작업 상태 변경 | `agent`, `state`, `progress`, `statusText`, `taskType?` (routine/project) |
| `file_transfer` | 파일 전달 | `from`, `to` |
| `artifact` | **신규** — 산출물 렌더링 | `agent`, `artifactType` (table/chart/copy/diagram), `content` |
| `decision_point` | **신규** — HITL 판단 요청 | `question`, `options[]`, `autoChoice`, `autoReason`, `timeout` (ms) |
| `routine_alert` | **신규** — 루틴 업무 알림 | `agent`, `text` |
| `result_ready` | 최종 결과물 팝업 | `resultType`, `content` |
| `publish_done` | SNS 배포 완료 팝업 | — |
| `sound` | **신규** — 알림음 재생 | `soundType` (notification/complete/alert) |

## 5. 데모 플로우 & Decision Points

### 5.1 전체 플로우

```
[Phase -1] Daily Routine — 에이전트들 각자 루틴 업무 중 (데모 시작 전)
     ↓ 사용자 토픽 입력
[Phase 0]  프로젝트 수주 — Alex "새 미션 도착" → 루틴 홀드 → 전환 모션
     ↓
[Phase 1]  킥오프 회의 — 전원 회의실 이동(걸어감) → 역할 분담
     ↓
[Phase 2]  병렬 작업 — 각자 데스크 복귀 → 산출물 실시간 렌더링
     ↓ ⚡ DP1: 타겟 범위 선택 (30초 타임아웃)
[Phase 3]  심화 작업 — 자체 판단 디테일 + 루틴 백그라운드 알림
     ↓ ⚡ DP2: 카피 톤 선택 (30초 타임아웃)
[Phase 4]  결과물 통합 — 파일 전달(걸어가서) → Alex 최종 컴파일
     ↓ ⚡ DP3: 예산 배분 선택 (30초 타임아웃)
[Phase 5]  최종 결과 — 한국어 마케팅 플랜 ResultPanel 팝업
     ↓
[Phase 6]  배포 & 완료 — CRM 연동 + SNS 배포 → 루틴 복귀 → 다음 액션
```

**총 예상 시간: 3분 30초 ~ 4분 (Decision Point 포함)**

### 5.2 Decision Points 상세

#### DP1 — 타겟 범위 (Phase 2 중간, ~25초 지점)

```
트리거: Kai의 리서치 초안 완료 직후
질문:   "팀장님, Kai의 리서치 결과 C레벨과 실무 매니저 두 세그먼트 모두 가능성이 있습니다. 어디에 집중할까요?"
선택A:  "C레벨 집중" — 딜 사이즈 크지만 전환 기간이 길어요 (예상 CAC ₩850,000)
선택B:  "실무 매니저 포함" — 볼륨 크고 빠르지만 단가 낮아요 (예상 CAC ₩320,000)
자동선택: B (근거: "예산 규모와 캠페인 기간 고려 시 B가 ROI 효율적이라 판단했습니다")
알림음:  notification
타임아웃: 30초
```

분기 영향:
- A 선택 시 → Nova가 "ABM(Account-Based Marketing) 퍼널"로 전환, Mia가 "C-Suite 전용 백서 톤" 카피, Rex가 "높은 CAC 감안 장기 LTV 모델"로 분석
- B 선택 시 → Nova가 "콘텐츠 리드 제너레이션 퍼널", Mia가 "실무자 공감 톤 카피", Rex가 "볼륨 기반 빠른 전환 모델"로 분석

#### DP2 — 카피 톤 (Phase 3 중간, ~15초 지점)

```
트리거: Mia의 카피 초안 2종 완료 직후
질문:   "팀장님, Mia가 카피 초안 2종을 준비했습니다. 어느 방향으로 갈까요?"
선택A:  데이터 신뢰 톤 — "검증된 ROI 342% 달성 기업의 선택"
선택B:  위기감 소구 톤 — "아직도 수작업으로 마케팅 하시나요?"
자동선택: A (근거: "B2B 타겟 특성상 신뢰 기반 톤이 전환율 1.8배 높다는 Kai의 데이터 근거입니다")
알림음:  notification
타임아웃: 30초
```

#### DP3 — 예산 배분 (Phase 4 직전, ~5초 지점)

```
트리거: Rex의 분석 완료 직후
질문:   "팀장님, Rex의 분석 결과 두 가지 배분 전략이 있습니다."
선택A:  "링크드인 올인" — 70:30 배분, 단기 리드 확보에 유리
선택B:  "2단계 전략" — 1차 리타겟팅 풀 확보(40%) 후 2차 집중 투입(60%)
자동선택: B (근거: "CAC 최적화 관점에서 2단계 전략이 장기 LTV에 유리합니다")
알림음:  notification
타임아웃: 30초
```

### 5.3 Decision Popup UI 명세

- 화면 정중앙, 반투명 배경 오버레이
- 카드 너비 520px, 둥근 모서리
- 상단: Alex 아바타 + 질문 텍스트
- 중단: 선택지 카드 (호버 시 예상 결과 미리보기 확장)
- 하단: 30초 원형 카운트다운 + "Alex가 자동 판단합니다" 안내 텍스트
- 알림음: Web Audio API — 짧은 bell sound (100ms)
- 선택 시: 선택한 카드 하이라이트 + 나머지 페이드아웃 + 0.5초 후 팝업 닫힘
- 타임아웃 시: Alex 아바타에 "판단 중..." 표시 → 1초 후 자동 선택 + 근거 채팅 로그

## 6. 시나리오 타임라인 상세

### Phase -1 — Daily Routine (데모 시작 전)

| 항목 | 값 |
|------|-----|
| **지속 시간** | 무한 (사용자 입력 대기) |

## 7. 2D 픽셀아트 비주얼 시스템

### 7.1 선택 근거

- 4/8 시연까지 시간 고려 시 3D보다 현실적
- 레트로 감성이 교육 데모에서 친근함 + 게임 느낌 유발
- 에셋 자체 제작 가능 (외부 의존 없음)
- 성능 이슈 없음 (CSS/Canvas 기반)

### 7.2 캐릭터 스프라이트

| 에이전트 | 기본 컬러 | 특징 |
|----------|----------|------|
| Alex | 파란 넥타이/셔츠 | 리더 느낌 |
| Kai | 노란 안경/후드 | 리서처 느낌 |
| Mia | 초록 머리띠/캐주얼 | 크리에이터 느낌 |
| Nova | 보라 블레이저 | 전략가 느낌 |
| Rex | 빨간 조끼/차트 장식 | 분석가 느낌 |

스프라이트 시트 구성 (캐릭터당):
- **크기:** 48×48px (OfficeScene 표시용) / 24×24px (BottomBar 축소용)
- **프레임:** 걷기(상하좌우 4방향 × 4프레임 = 16), 앉기(2), 타이핑(4), idle(2), 완료(2) = **총 26프레임**
- **구현:** CSS sprite-sheet + `requestAnimationFrame` 또는 Canvas `drawImage`

### 7.3 오피스 타일맵

- **스타일:** 탑다운(top-down) 뷰, 약간의 아이소메트릭 느낌
- **타일 크기:** 32×32px
- **구성 요소:** 바닥 타일, 책상(5개, 모니터 포함), 회의 테이블(1개), 의자, 화분, 화이트보드, 복도
- **맵 크기:** 약 20×12 타일 (640×384px → OfficeScene 영역에 맞춰 스케일)

### 7.4 에셋 조달 방식

**우선순위 1: 코드 생성 (외부 의존 없음)**
- 단색 실루엣 + 컬러 포인트 방식
- Canvas API로 직접 그리기
- 장점: 즉시 구현, 스타일 통일, 수정 자유

**우선순위 2: 무료 픽셀아트 팩**
- itch.io "Modern Office" 타일셋 (무료/CC0)
- 캐릭터는 "Tiny RPG Character" 스프라이트 커스텀
- 라이선스 확인 필요

### 7.5 애니메이션 시스템

```
이동 (A → B):
1. 현재 위치에서 "일어서기" (앉기 → idle 전환)
2. A* 또는 직선 경로 계산
3. 걷기 스프라이트 재생 (방향에 따라 상/하/좌/우)
4. 도착 후 "앉기" (idle → 앉기 전환)
5. 앉은 상태에서 "타이핑" (작업 시작 시)

파일 전달 (에이전트 A → 에이전트 B):
1. A가 일어남
2. A가 B 데스크까지 걸어감 (📄 아이콘 머리 위에)
3. B 데스크 도착 → 📄 전달 애니메이션
4. A가 자기 데스크로 복귀

프레임 레이트: 8fps (픽셀아트 표준)
```
